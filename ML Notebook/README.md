# ML Notebook

This folder contains exploratory notebooks for the machine-learning side of the project.

## Contents

- `Capture TCP Stream to MP4.ipynb`: Connects to the ESP32 TCP stream and saves incoming JPEG frames into an `.mp4` recording.
- `Collecting Data .ipynb`: Notebook used while gathering training or validation data.
- `Testing video stream.ipynb`: Notebook for experimenting with the incoming frame stream.
- `Training notebook.ipynb`: Notebook for model training and related experiments.

## Intended Use

These notebooks are for iteration and investigation, not for the production runtime path.

Use them when you need to:

- inspect incoming data
- prototype preprocessing or training changes
- validate model ideas before moving code into regular Python modules

## Notes

- Notebook outputs and local paths may be machine-specific.
- `.ipynb_checkpoints/` is generated by Jupyter and is not hand-maintained project source.
- The TCP stream notebooks assume the repo's current frame format: 4-byte big-endian JPEG length followed by JPEG bytes.
- Once a workflow is stable, move the important logic into regular Python modules or documented scripts.
